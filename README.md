# Trabajo Práctico N° 2

## Criterios de evaluación

Crear un nuevo repositorio en GitHub para esta actividad y configurarlo para que funcione como sitio publicado mediante GitHub Pages.

Inicializar el proyecto con Git y registrar el proceso de desarrollo mediante commits significativos.

Desarrollar 3 páginas web HTML relacionadas entre sí mediante enlaces de navegación:

index.html

destacados.html

listado.html

La página principal index.html debe incluir enlaces hacia destacados.html y listado.html.

Tanto destacados.html como listado.html deben incluir un enlace de regreso a index.html.

En el conjunto del sitio se debe evidenciar el uso de:

modelo de caja (margin, padding, border, width, height, etc.);

JavaScript básico con variables, condicionales, funciones y arreglos;

métodos de arreglos como map, forEach, filter u otros equivalentes;

manipulación del DOM;

manejo de eventos.

No se debe solicitar ni utilizar formularios en esta actividad.

Los ejercicios no deben ser complejos. Se espera una resolución simple, clara y funcional.

Resolver los siguientes 3 ejercicios concretos:

Ejercicio 1 - Página principal index.html: menú de meriendas

Crear un arreglo con al menos 5 meriendas o colaciones.

Mostrar en pantalla la lista de meriendas usando JavaScript y un método de arreglos como map o forEach.

Incluir un botón que, al hacer clic, muestre cuántos elementos tiene el arreglo en un texto del DOM.

La página debe presentar los bloques de contenido usando propiedades del modelo de caja.

Ejercicio 2 - Página destacados.html: tarjetas seleccionables

Crear al menos 4 tarjetas con información breve (por ejemplo: películas, juegos, canciones, comidas o lugares).

Al hacer clic sobre cada tarjeta, esta debe cambiar su aspecto visual para indicar que quedó seleccionada.

La selección debe resolverse usando classList (add, remove o toggle).

Debe existir un texto o contador en la página que informe cuántas tarjetas están seleccionadas.

Ejercicio 3 - Página listado.html: filtrado simple de elementos

Crear un arreglo con al menos 6 elementos y mostrarlos en pantalla como bloques o tarjetas.

Incluir dos botones: uno para mostrar todos los elementos y otro para mostrar solo una categoría o grupo definido por el estudiante (por ejemplo, "solo frutas", "solo juegos", "solo pendientes").

El cambio de vista debe resolverse con JavaScript, manipulando el DOM y utilizando un método de arreglos como filter.

La presentación visual debe apoyarse en el modelo de caja.

No se permite el uso de librerías o frameworks externos de CSS o JavaScript.
